## 3. 系统架构

### 3.1 服务清单

引用：`docs/ARCHITECTURE.md`。

### 3.2 核心链路

```text
登录 → 商品 → 购物车 → 订单 → 锁库存
→ 支付消息 → 更新订单 → 扣减库存
```

### 3.3 架构取舍

说明以下决策：

- 为什么保留 13 个服务但不继续新增；
- 为什么只对核心 Feign 调用增加降级；
- 为什么支付采用模拟和异步消息；
- 为什么 Kubernetes 不是核心交付路径；
- 如何控制项目规模和技术复杂度。
