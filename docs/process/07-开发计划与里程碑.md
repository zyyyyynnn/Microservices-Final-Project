## 1. 团队分工

团队固定为 5 人。最终姓名由小组填写，按以下职责划分：

| 角色 | 主要职责 |
|---|---|
| 成员 1：架构与网关 | 项目基线、Gateway、Nacos、公共模块 |
| 成员 2：用户与商品 | Auth、User、Product、Search |
| 成员 3：交易链路 | Cart、Order、Inventory、Seata |
| 成员 4：支付与异步 | Pay、Message、RocketMQ、Seckill |
| 成员 5：测试与交付 | Postman、JMeter、部署、报告、答辩 |

分工不代表模块完全隔离。核心链路联调、测试和答辩由全体成员共同负责。

## 2. 里程碑阶段

### M1：基线与标准建立

- 确立技术基线：Java 21、Spring Boot 3.2.4、Spring Cloud Alibaba 2023.0.1.0、Seata 2.0.0。
- 冻结项目范围与不做清单。
- 建立项目标准、编码规范、文档优先级。
- 完成服务划分与端口规划。

### M2：核心链路实现

- 实现 Gateway 路由与 JWT 鉴权。
- 实现 Auth、User、Product、Cart、Order、Inventory、Pay 服务骨架与核心写流程。
- 打通普通下单链路（订单→商品→库存）。
- 完成数据库初始化脚本与种子数据。

### M3：技术亮点与异步链路

- 引入 Seata 2.0.0，验证订单-库存一致性。
- 引入 RocketMQ，完成支付结果异步消费与库存扣减。
- 引入 Sentinel，完成秒杀限流与库存熔断。
- 完成 Redis 购物车与秒杀库存原子操作。
- 完成 Elasticsearch 商品搜索索引。

### M4：前端产品化

- 完成前台商城与后台管理页面。
- 前端请求统一经 Gateway `/api/v1/**`。
- 完成 loading/empty/error/disabled/success 状态覆盖。
- 完成桌面端与移动端响应式适配。

### M5：测试与验收

- 完成 Postman/Newman 接口测试集合。
- 完成 JMeter 搜索、订单、秒杀负载与压力测试。
- 完成 Nacos 热更新、Sentinel 限流、Seata 回滚、重复消息幂等异常测试。
- 完成 Gateway 单元测试（RANDOM_PORT 真实链路）。
- 完成运行时回归与启动状态日志验收。

### M6：交付与答辩

- 完成最终页面截图矩阵。
- 完成图表交付（Mermaid/SVG/PNG）。
- 完成答辩材料（PPT、视频脚本、关键代码导读）。
- 完成最终验收报告与边界声明。

## 5. 修改流程标准

任何后续修改遵循：

```text
先修改标准与需求
  → 再修改设计与接口
  → 再修改代码与配置
  → 最后补测试证据
```

不得只修改代码而不更新相关文档，也不得把规划能力描述为已实现能力。
