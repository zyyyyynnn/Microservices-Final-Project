## 1. 数据库初始化脚本

MallCloud 当前只支持基于 `db/init/` 的新环境初始化，不再维护旧数据库环境升级脚本。

| 路径 | 用途 | 适用场景 |
|---|---|---|
| `db/init/00-create-databases.sql` | 创建 8 个业务库与全部表结构 | 新 MySQL 容器首次初始化 |
| `db/init/seed.sql` | 初始化演示数据 | 新 MySQL 容器首次初始化 |
| `db/demo/demo-data-20260611.sql` | 演示用补充数据 | 演示场景可选 |

业务库清单：

| 数据库 | 所属服务 | 主要表 |
|---|---|---|
| `mall_auth` | mall-auth | `sys_user_auth`、`undo_log` |
| `mall_user` | mall-user | `user`、`address`、`undo_log` |
| `mall_product` | mall-product | `category`、`spu`、`sku`、`spu_attr`、`spu_img`、`undo_log` |
| `mall_inventory` | mall-inventory | `stock`、`stock_log`、`undo_log` |
| `mall_order` | mall-order | `order_info`、`order_item`、`order_log`、`undo_log` |
| `mall_pay` | mall-pay | `pay_record`、`refund_record`、`undo_log` |
| `mall_seckill` | mall-seckill | `seckill_activity`、`seckill_order`、`undo_log` |
| `mall_seata` | Seata Server | `global_table`、`branch_table`、`lock_table`、`distributed_lock` |

表结构权威 DDL 以 `db/init/00-create-databases.sql` 为准；表结构设计说明见 [05-数据库设计文档.md](05-数据库设计文档.md)。

## 2. 执行规则

- 新环境使用 `db/init/`；
- MySQL Docker 首次初始化时自动执行 `db/init/` 下脚本；
- 不要在已有数据的数据库中重复执行 `seed.sql`；
- 如需重建演示数据，应备份后重建数据库容器；
- `seed.sql` 不是幂等脚本，无 `-Force` 时 `init-db.ps1` 默认拒绝执行，避免误重置已有数据。

执行命令：

```powershell
pwsh .\scripts\init-db.ps1 -Force
```

```sql
SELECT COUNT(*) FROM mall_user.user;
SELECT COUNT(*) FROM mall_product.spu;
SELECT COUNT(*) FROM mall_product.sku;
SELECT COUNT(*) FROM mall_inventory.stock;
SELECT COUNT(*) FROM mall_seckill.seckill_activity;
```

预期分别为 10、5、7、7、3。

容器内快速验证：

```powershell
docker exec mall-mysql mysql -uroot -proot -e "SELECT COUNT(*) AS users FROM mall_user.user; SELECT COUNT(*) AS spu FROM mall_product.spu; SELECT COUNT(*) AS sku FROM mall_product.sku;"
```

## 6. 秒杀压测数据准备

秒杀正式压力测试前需准备可重复的多用户数据：

```powershell
pwsh .\scripts\prepare-seckill-jmeter.ps1 -ActivityId 9001 -SkuId 99003 -TotalStock 100 -UserCount 100
```

该脚本会：

- 准备 `jmeter_seckill_1..N` 测试用户，默认密码 `123456`；
- 创建或重置专用活动 `9001`，对应专用压测 `skuId=99003`（从种子 SKU `9003` 复制，总库存 100、每用户限购 1）；
- 清理该活动下游秒杀订单、订单明细、库存流水、SKU 库存；
- 清理 Redis 中该活动的库存缓存和限购 Key；
- 默认通过宿主机 MySQL 连接执行，后端连接 Docker MySQL 时可使用 `-MysqlMode docker`。

秒杀 Newman 用例使用 `zhangsan` 和当前进行中的种子活动，正式回归前必须清理该测试用户对目标活动的限购状态，或重置 Redis 与数据库测试数据。不得通过放宽秒杀限购断言来规避重复执行问题。

## 7. 联调边界说明

- 数据库种子数据中文字段存在编码异常，业务链路不受影响，后续可独立修复 `seed.sql`；

- `full` Profile 外部基础设施能力按本地真实复跑记录描述，不扩大为高可用部署；
