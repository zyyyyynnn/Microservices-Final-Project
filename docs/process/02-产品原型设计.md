## 1. 产品目标

MallCloud 是面向微服务课程期末大作业的演示与测试系统。产品设计目标是把当前后端微服务能力组织成一条可演示、可测试、可展示的用户体验链路：

```text
登录 / 注册
  → 商品浏览 / 分类检索
  → 商品搜索与详情查看 (选择 SKU)
  → 购物车管理 (添加、勾选、删除、改数)
  → 确认结算与订单创建 (库存锁定)
  → 模拟支付发起 (创建支付流水)
  → 支付消息分发与处理 (订单状态与库存更新)
  → 查询订单与支付记录 (演示闭环)
```

设计约束：

- 前台不暴露任何后端开发及服务联调状态字眼；
- 商家及管理员能够在后台浏览关键统计指标并执行订单发货及商品基础维护；
- Gateway 鉴权、服务隔离、RocketMQ、Seata、Sentinel 限流和秒杀并发等后端核心技术在前端有明确承载和入口；
- 所有未通过最终网页端人工复验和视觉验收的页面与状态一律标记为“待网页端复审，待视觉验收”，不得写成已通过。

## 2. 目标用户与角色

| 角色 | 产品目标 | 权限范围 | 状态 |
|---|---|---|---|
| 游客 | 浏览公共商品信息、分类展示与检索搜索 | 公共商品详情、分类树、搜索结果、热门搜索词、账户注册与登录 | 能力已定义 |
| 普通用户（`USER`） | 体验完整的购物流程及核心订单交易链路 | 个人信息修改、地址簿管理、购物车管理、订单创建、模拟支付、秒杀参与 | 能力已定义 |
| 商家（`MERCHANT`） | 课程演示所需的商品和订单基础维护 | 后台订单列表、发货、商品列表浏览及商品上下架 | 权限边界待确认，后台登录待确认 |
| 管理员（`ADMIN`） | 后台全部指标浏览与核心后台演示操作 | 后台看板、订单及商品全局管理 | 权限边界待确认，后台登录待确认 |

演示账号统一密码为 `123456`，仅限本地测试使用。

## 3. 信息架构

当前微商城前端（`mall-frontend`）通过路由划分前台商城与后台管理功能区：

```text
MallCloud 前端应用
├── 公共服务区
│   ├── 注册页 (RegisterView)
│   ├── 登录页 (LoginView)
│   ├── 首页 / 商品推荐 (HomeView)
│   ├── 商品详情 (ProductDetailView)
│   ├── 商品搜索 / 热词 (SearchView)
│   └── 404 错误页 (NotFoundView)
├── 用户中心 (受限访问)
│   ├── 账户资料与头像 (AccountView)
│   └── 收货地址管理 (AccountView)
├── 购物车系统 (受限访问)
│   └── 购物车明细与结算入口 (CartView)
├── 订单与结算流 (受限访问)
│   ├── 订单结算确认 (CheckoutView)
│   ├── 订单详情与物流状态 (OrderDetailView)
│   ├── 模拟支付收银台 (PayView)
│   └── 秒杀活动大厅 (SeckillView)
└── 后台管理系统 (受限访问，仅限管理员/商家)
    ├── 数据指标看板 (AdminView)
    ├── 后台订单列表与发货 (AdminView)
    └── 后台商品列表与维护 (AdminView)
```

## 4. 核心用户流程

### 4.1 认证与登录上下文

1. 游客在 `/login` 提交表单，经 Gateway 转发至 `mall-auth` 进行校验。
2. 登录成功后，前端 Pinia Store 存储 `accessToken`、`refreshToken` 和 `userInfo`。
3. 后续所有受限请求通过 Axios 拦截器在 Header 携带 `Authorization: Bearer <token>`。
4. 退出登录时调用 `/api/v1/auth/logout`，并清空本地存储，重定向回登录页。

### 4.2 购物车交易流

1. 用户在商品详情页选择具体 SKU 属性和购买数量。
2. 点击“加入购物车”触发请求，后台将商品加入 `mall-cart`。
3. 购物车页面拉取列表，允许用户修改数量、单项勾选、一键删除等，最终汇总计算总额。

### 4.3 结算与下单

1. 用户从购物车或商品详情点击结算，进入 `/checkout`。
2. 页面拉取收货地址列表，用户确认收货地址并提交订单。
3. `mall-order` 接收请求，调用商品服务验证价格，调用 `mall-inventory` 锁定库存，创建成功后返回 `orderNo` 并跳转至订单详情页。

### 4.4 模拟支付与回调

1. 用户从订单详情进入 `/pay/:orderNo`。
2. 点击“确认支付”触发模拟收银台创建支付单（`mall-pay`）。
3. 页面展示“确认模拟支付回调”的操作按钮，点击后模拟发送支付成功消息（`PAY_RESULT`）。
4. RocketMQ 分发消息：`mall-order` 更新订单状态为已付款，`mall-inventory` 正式扣减库存，用户可查询到最新的订单详情。

### 4.5 秒杀高并发交易

1. 用户进入 `/seckill` 活动大厅，获取当前的秒杀活动列表。
2. 活动开始后，点击“立即抢购”发起秒杀异步请求，后台写入 Redis 并返回 `requestId`。
3. 前端获取 `requestId` 后进入轮询等待状态，直到获取到成功或失败的最终结果。

## 5. 页面与路由

| 页面名称 | 路由路径 | 目标角色 | 核心依赖 API |
|---|---|---|---|
| 登录页 | `/login` | 游客 | `POST /api/v1/auth/login` |
| 注册页 | `/register` | 游客 | `POST /api/v1/users/register` |
| 首页 | `/` | 游客/用户 | `GET /api/v1/categories/tree` |
| 商品详情页 | `/products/:id` | 游客/用户 | `GET /api/v1/products/{id}` |
| 商品搜索页 | `/search` | 游客/用户 | `GET /api/v1/search/products` |
| 购物车页 | `/cart` | 用户 | `GET /api/v1/carts` |
| 个人账户页 | `/account` | 用户 | `GET /api/v1/users/me`、地址 CRUD |
| 确认结算页 | `/checkout` | 用户 | `POST /api/v1/orders` |
| 订单详情页 | `/orders/:orderNo` | 用户 | `GET /api/v1/orders/{orderNo}` |
| 模拟支付页 | `/pay/:orderNo` | 用户 | `POST /api/v1/pay/create` |

## 6. 页面级交互要求

### 6.1 前台商城页面

- **首页 (HomeView)**：首屏展示“交易链路（演示流程）”步骤指引卡片；核心频道支持点击一键切换搜索；推荐商品列表及限时秒杀展示区采用规范网格排列。
- **商品详情 (ProductDetailView)**：完整展示主图、商品名、SPU/SKU 选择器、价格、库存以及描述；切换 SKU 时价格、库存和购买数量限制瞬时刷新；库存为 0 或下架的 SKU 对应按钮禁用。
- **搜索页 (SearchView)**：结果网格展示，无匹配时展示统一 Empty 空态，并展示“推荐热词”引导重试。
- **购物车页 (CartView)**：勾选行、改变数量、单项删除均响应式联动底部结算悬浮条；修改数量时该行进入提交锁定状态。
- **账户页 (AccountView)**：整合用户资料卡与收件地址卡；地址列表提供新增/编辑/删除表单。

### 6.2 后台管理页面

- **后台管理 (AdminView)**：看板指标区三格栅格呈现订单数、商品数、销售额；数据缺失时显示 `—` 兜底；商品列重构为商品图文卡片；发货区域使用横向排版，响应式适配移动端。

## 7. UI 状态规范

| 状态 | 要求 |
|---|---|
| Loading | 页面级使用骨架屏或静态占位，禁止旋转加载动画；操作按钮点击后立即 `:loading` 或 `:disabled` 并显示“处理中” |
| Empty | 统一 `<el-empty>` 结构；订单/商品/秒杀空态描述统一 |
| Error | 统一 `PageState` 或 `el-alert` 提示并提供“重试”；用户可见文案不得出现技术错误码、异常名和微服务名称 |
| Disabled | 通过 token 化背景、边框、文本色、cursor 和 disabled 属性共同表达；严禁仅靠 `opacity` 表达 |
| Success | Element Plus Message 提示，按钮恢复可用，文案客观明确 |

数据兜底：字段为 null、undefined、NaN、空字符串或接口未返回时展示 `—`；合法 0 值正常展示。

## 9. 响应式行为

| 断点 | 核心行为 |
|---|---|
| 桌面端大屏 (>= 1440px) | 最大宽度 `1440px` 居中，两侧 gutter `32px` |
| 小屏笔记本 (1024px ~ 1366px) | gutter 降为 `24px`，导航完整展开，推荐商品网格 4-5 列自适应 |
| 平板端 (768px ~ 1023px) | 首页 Hero 折叠单栏，Header 触发折叠，购物车结算栏悬浮固定底部 |
| 移动端窄屏 (< 768px，最小 390px) | gutter `16px`，表格横向滚动或流式卡片，Dialog/Drawer 自适应不超出视口 |
