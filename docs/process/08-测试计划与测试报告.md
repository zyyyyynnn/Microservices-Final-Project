# MallCloud 测试计划与测试报告

> 文档版本：v1.0
> 事实来源：[docs/test/README.md](../test/README.md)、[FINAL_REPORT.md](../documents/FINAL_REPORT.md)

---

## 1. 测试目标

- 验证核心交易主链路经 Gateway 可执行；
- 验证 Spring Cloud Alibaba 核心组件（Nacos、Gateway、OpenFeign、Seata、RocketMQ、Sentinel、Redis、Elasticsearch）正确使用；
- 验证服务治理、限流熔断、分布式事务、消息异步等关键技术能力；
- 形成可重复执行的接口测试、负载测试、压力测试和异常测试证据；
- 失败项如实记录，不隐藏、不写成通过。

---

## 2. 测试范围

| 类型 | 范围 | 工具/方式 |
|---|---|---|
| 接口测试 | 认证、用户、商品、搜索、购物车、订单、库存、支付、秒杀、后台 | Postman/Newman |
| 服务治理 | Nacos 注册/下线、Gateway 路由、JWT 白名单/错误 Token | Postman + Nacos 控制台 |
| 负载测试 | 商品查询 50/150 用户、创建订单 | JMeter |
| 压力测试 | 秒杀逐级加压至 500 用户 | JMeter |
| 异常测试 | 下游宕机、Sentinel 限流/熔断、Nacos 热更新、库存边界、Seata 回滚、重复消息幂等 | 手动 + 脚本 |
| 前端验收 | 桌面端/移动端页面状态覆盖 | 浏览器 + 截图 |
| 单元测试 | Gateway JWT、公共模块 | Maven test |

| 测试 | 命令 |
|---|---|
| Newman 接口测试 | `pwsh .\scripts\run-newman.ps1` |

## 6. 测试结果可信度要求

禁止：

- 手工修改 JTL 或 HTML 结果；
- 用预估值代替 P95；
- 只截成功请求而隐藏失败请求；
- 省略测试机器配置、JDK 版本和 Commit；
- 把未执行脚本标记为通过；
- 使用固定 Token 假装登录链路有效；
- 把测试资产建立写成测试通过。

允许并建议如实记录：性能目标未达到、服务瓶颈、兼容问题、功能尚未验证、阈值需调整、资源限制影响结果。
