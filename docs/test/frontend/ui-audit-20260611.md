# MallCloud UI 视觉复审报告

## 1. 审查结论
**有条件通过**。
当前前端代码已完成基本的页面路由切分与重构，但仍残留明显的“毛坯房感”，且在 CSS 实现上与 `DESIGN.md` 中的“零动效”等约束存在直接冲突。需在进入下一阶段视觉升级前优先清理 P0 和 P1 问题。

## 2. 审查环境
- **启动命令**: `cmd /c start-all.bat --profile core --no-build --skip-frontend --no-pause`
- **前端地址**: `http://localhost:5173`
- **后端 Profile**: `core`
- **浏览器**: Puppeteer (Chromium / Headless)
- **视口尺寸**: 1440x900, 1366x768, 1024x768, 768x1024, 430x932, 390x844, 375x812

## 3. 总体判断
**是否仍有毛坯房感**：是。主要体现在纯白背景叠加大面积无层次白色卡片、调试台风格的原生组件直接暴露，以及由于 CSS 冲突导致的悬浮跳动 Bug。

**最主要的 5 个视觉问题**：
1. **强制去动效与 Transform 冲突**：全局 `reset.css` 去除了 transition，但 `app.css` 保留了 hover transform，导致商品卡片 hover 时发生 2px 的生硬跳变。
2. **移动端横向溢出**：`.app-nav` 使用 `100vw` 宽度直接导致移动端出现双横向滚动条，严重破坏页面结构。
3. **主导航沦为功能清单**：顶部导航暴露了后台、技术演示、购物车、秒杀等所有入口，且中英文混用 (Guest / Sign In)，缺乏产品化包装。
4. **视觉层级扁平缺失焦点**：卡片、背景和弹窗均使用极简白色和浅边框，导致内容缺乏视觉焦点，“品牌蓝”由于被 Element Plus 直接映射为 primary，在按钮上使用泛滥。
5. **后台/技术页调试感过重**：技术演示页直接使用 `<pre>` 标签做 JSON dump，后台订单使用极简表格配合 Inline Form 发货，看起来更像是 API 调试台。

**是否建议进入改造**：是。

## 4. P0 阻断问题

### 1. 零动效约束违背与悬浮跳动 Bug
- **出现页面**：全局（商品列表页、类目页、活动选项）
- **证据文件**：`app.css` (L79, L262, L290, L379) 和 `reset.css` (L38)
- **原因**：`reset.css` 中使用了 `transition: none !important;` 阻断了缓动效果，但 `app.css` 中 `.category-item:hover` 和 `.product-card:hover` 依旧写入了 `transform: translateY(-2px);`。这导致原本期望的缓动上浮变成了点击般的突兀生硬跳动，极度降低质感。
- **建议方向**：彻底删除 `app.css` 中的 `transition` 和 `transform` 相关属性，用纯静态的边框颜色加深（如 `var(--color-border-strong)`）代替位移。
- **是否需要修改 DESIGN.md**：否，严格执行现有的“零动效”与“不使用 hover 形变”规则。

### 2. 移动端横向溢出
- **出现页面**：全局（移动端视口 < 640px）
- **证据文件**：`app.css` (L483)
- **原因**：媒体查询中 `.app-nav` 设置了 `width: 100vw;`。由于没有考虑父级 `.app-header` 的 padding 和系统滚动条宽度，直接导致移动端出现难以操作的双向横向滚动。
- **建议方向**：修改为 `width: 100%;` 并利用 Flexbox 溢出控制，或将导航收纳为汉堡菜单/底部 Tab。
- **是否需要修改 DESIGN.md**：否。

## 5. P1 高优先级问题

### 1. 导航层级与中英文混用
- **出现页面**：Header 导航
- **证据文件**：`App.vue`
- **原因**：`App.vue` 将后台、技术演示页与常规核心电商路径平铺在一起，同时 Session 状态展示混合了 "Guest", "Sign In", "Sign Out" 与中文字体，破坏了产品感。
- **建议方向**：将后台和技术页收纳进特定的折叠菜单或 Footer 演示区，将 Guest/Sign In 替换为纯中文语境的“登录/注册”。
- **是否需要修改 DESIGN.md**：否。

### 2. 页面“API 测试台”化
- **出现页面**：技术演示页 (`/tech`)、后台 (`/admin`)
- **证据文件**：`TechView.vue`, `AdminView.vue`
- **原因**：技术演示页大量使用 `<pre class="json-box">` 原生展示，后台发货与商品管理使用默认无阴影的 `el-table` 和顶部 Inline Input，缺少典型的仪表盘数据可视化设计。
- **建议方向**：后台可引入骨架屏和轻量图表卡片增强仪表盘感；技术演示可使用格式化树状图组件 (如 vue-json-viewer) 代替直接 Dump。
- **是否需要修改 DESIGN.md**：否。

## 6. P2 普通优化

### 1. Element Plus 默认痕迹过强
- **出现页面**：所有表单与提示组件
- **证据文件**：`element-theme.css`
- **原因**：只修改了圆角和边框颜色，但 `el-alert`, `el-tag`, `el-button` 等组件的填充逻辑依然是典型的 Element 蓝。Primary Button 被滥用在次要操作上。
- **建议方向**：限制 primary 按钮在每个页面仅有一个 (核心 Action)，其他均使用 `plain` 或次级按钮。

### 2. 禁用态缺乏辨识度
- **出现页面**：Checkout 提交按钮, 秒杀按钮
- **证据文件**：`element-theme.css`
- **原因**：`opacity: 1` 配合 `var(--color-surface-muted)`。由于背景过浅，它和常规未激活的元素视觉差距过小。
- **建议方向**：加深禁用态的底色，或为禁用态引入明显的 disabled cursor (not-allowed) 和斜纹背景。

## 7. 页面逐项问题

- **首页 / 搜索 / 商品详情**：卡片大面积白底（与 `body` 白色无对比度），导致卡片看起来就是漂浮的几根线条。建议卡片 hover 时加深阴影和边框。
- **购物车 / 结算**：表格操作列过于紧凑，`el-table` 斑马纹/边框设计在响应式下过于拥挤。
- **技术演示 / 后台**：缺乏 Dashboard 质感，像管理系统脚手架默认生成的列表。
- **用户状态**：`.user-status` 使用 `Consolas` 字体，虽然本意是极简，但在电商 Header 出现等宽字体非常奇怪。

## 8. 响应式问题
- **430x932 (移动端)**: Header 横向溢出（由于 `100vw`）；Checkout 表单字段挤在一起。
- **1024x768 (窄桌面)**: `app-main` 固定宽度机制导致内容部分几乎贴边，两边留白不足。

## 9. 状态视觉问题
- **Empty**: 列表为空时大量展示大面积留白和极简的 `el-empty`，缺乏可执行的下一步（例如：购物车为空时应有去逛逛的巨大按钮）。
- **Loading**: 当前为 Element Plus 默认的转圈遮罩，页面加载时瞬间白屏。
- **Success/Error**: 操作提示通常只是右侧弹出一个顶部 Message，对于“秒杀成功”这种核心链路，缺少页面级的成功反馈 (Success State Page)。

## 10. 设计系统问题
- **字体**: 中英文混杂导致 Lora/Noto Serif SC 与系统无衬线字体交替出现，破坏了复古感。
- **颜色/阴影**: `DESIGN.md` 中写明了 `--shadow-sm` 和 `--shadow-md` 的复杂多层 Airtable 蓝调阴影，但大部分区域都被取消 (`box-shadow: none !important;`)。
- **间距**: 列表卡片间的 `gap` 有时与卡片内部 `padding` 不匹配。

## 11. 与 DESIGN.md 冲突点
1. **文件**: `app.css`  
   **选择器**: `.nav-link`, `.category-item`, `.product-card`, `.activity-card` 等  
   **原因**: 使用了 `transition` 和 `transform`。完全违背“禁止动画和缓动”、“禁止使用 hover 形变”的约束。
2. **文件**: `app.css`  
   **选择器**: `.user-status`  
   **原因**: 违背“数据、订单号、SKU等使用等宽字体”的约束，用户状态不属于数据或代码。

## 12. 建议改造范围
1. 彻底清除 `app.css` 所有 `transition` 和 `transform`，保留 `border-color` 改变或 `box-shadow` 改变作为悬停反馈。
2. 修复 `app.css` 中的 `100vw` 响应式溢出 Bug。
3. 重构 `App.vue` 的顶部导航，隐藏内部测试路由，统一为中文。
4. 在关键成功态（支付完成、秒杀结果）补充占位明确的成功视觉反馈，而不是单纯显示一段 JSON。

## 13. 下一轮视觉升级验收标准
- 核心操作必须遵守零动效，hover 反馈仅限变色、阴影加深或边框变粗，无任何像素跳跃。
- 移动端和任意视口均无横向溢出。
- 无测试用和开发调试用的纯英文标识直接暴露在 C 端主导航。
- 后台与技术页必须包装为可用组件，彻底告别 raw JSON 调试台质感。
