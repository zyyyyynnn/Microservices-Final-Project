## 常见追问准备

### 1. 为什么说不是生产级系统？

因为当前验证目标是课程项目本地演示。Gateway 已验证 JWT、internal 阻断和请求头净化，但没有 mTLS、服务网格、生产密钥轮换和高可用集群验证。

### 2. 为什么 Admin 没有用户管理截图？

当前前端路由只有 `/admin`，组件 `AdminView.vue` 只包含 Dashboard、后台订单和后台商品区块。没有 `/admin/users` 路由和用户管理组件，因此最终交付明确记录缺失，不伪造截图。

### 3. 为什么需要 RANDOM_PORT Gateway 测试？

早期 MOCK 链路只能验证过滤器组合逻辑，不能证明真实 Web Server 下请求头大小写、路由转发和过滤器顺序。RANDOM_PORT 测试启动真实 server，用 mock downstream 替代真实下游，覆盖 internal 阻断、普通路径转发、请求头净化和 Authorization 不绕过。

### 4. Seata 的作用是什么？

创建订单时订单服务需要写订单表，同时库存服务需要锁定库存。Seata 用于把这两个本地事务纳入同一全局事务，异常时订单写入和库存锁定可一起回滚。

### 5. RocketMQ 在项目里解决什么问题？

支付成功后不直接同步改所有服务状态，而是发布支付结果消息。订单服务消费后标记已支付，库存服务消费后确认扣减，体现异步解耦和最终一致性。

### 6. 秒杀是否证明能支撑生产高并发？

不能。当前秒杀验证证明课程数据和本地环境下入口限流、库存不足、已结束和结果轮询链路可运行，不代表生产容量或商用压测结论。
